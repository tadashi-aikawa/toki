# セッションライフサイクル

## ssコマンドによるステータス管理

セッション中、以下のタイミングで `ss` コマンドを実行してステータスを管理する。

### セッション開始時

```
ss start "<依頼の要約>"
```

- `<依頼の要約>`: 先頭に依頼のタイプを表す絵文字をつける
- `/task` でセッションを開始した場合は、スキル内で実行済みのため省略する

### 応答の開始前

```
ss progress <進捗率(0~1)> <フェーズ名>...
```

- 応答中でも進捗率やフェーズに変化があれば都度実行する

### 応答の完了後

```
ss wait
```

## /task セッション固有のルール

`/task` でセッションを開始した場合、以下の追加ルールが適用される。

- `ss progress` 実行時 → `/task progress <id>` も実行する
- `ss wait` 実行時 → `/task wait <id>` も実行する
- `/task done <id>` はユーザーが明示的に指示した場合のみ実行する。自己判断で完了にしてはならない
