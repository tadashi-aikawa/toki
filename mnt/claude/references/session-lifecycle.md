# セッションライフサイクル

セッション中は以下のタイミングで処理を実行する。

## タスクノート紐づけの判定

各タイミングで `ss task-id` を都度実行し、値が取得できた場合はワークスペースがタスクノートと紐づいていると判断する。取得できなかった場合は紐づいていないと判断する。

## セッション開始時

### タスクノートと紐づいている場合

何もしない

### タスクノートと紐づいていない場合

```
ss start "<依頼の要約>"
```

## 応答の開始前

応答中でも進捗率やフェーズに変化があれば都度実行すること。

### タスクノートと紐づいている場合

1. `ss progress <進捗率(0~1)> <フェーズ名>` を実行
2. `ss task-id` でタスクノート紐づき確認
3. task-progress スキルを id 指定で使う

### タスクノートと紐づいていない場合

```
ss progress <進捗率(0~1)> <フェーズ名>...
```

## 応答の直前

### タスクノートと紐づいている場合

1. `ss wait` を実行
2. `ss task-id` でタスクノート紐づき確認
3. task-wait スキルを id 指定で使う

### タスクノートと紐づいていない場合

```
ss wait
```
